# report-web

발표 자료 기획 → MD 규격화 → 파트 분할 웹앱 구현 → 디자인 피드백 → 게시까지 한 흐름으로 진행하는 5단계 워크플로우.

## 핵심 원칙

- **정보 전달력 > 소개·장식** — 표·카드·숫자 위주, 문체적 꾸밈 최소화
- **PPT 타이포** — 핵심 수치·인용문은 `clamp(48px~96px)` 대형 강조
- **16:9 고정** — 모바일 고려 X, 데스크톱 발표·공유 전제
- **파트별 분할** — 편집·보강이 섹션 단위로 쉬워야 함. 단일 index.html 지양
- **전역 CSS 규칙(특히 `.child + .child { margin }`)은 flex/grid의 `gap`과 충돌** — 간격은 항상 부모 컨테이너의 `gap`으로 부여

## 5단계 워크플로우

각 단계는 사용자와 **대화로 확정**한 뒤 다음으로 넘어간다. 임의로 건너뛰지 말 것.

---

### Phase 1. 발표 자료 기획

요구사항 수집 + 대략적 챕터 구성까지 합의한다.

#### 1-1. 요구사항 질문

사용자에게 아래를 순차적으로 확인한다. 이미 대화 맥락에 답이 있으면 건너뛴다.

1. **대상 독자** — 신규 직원? 경영진? 외부 고객? 전 직원 공통?
2. **원본 자료** — CSV / 회의록 / 노션 URL / 스프레드시트 / 메모
3. **주제·목적** — 한 문장 요약
4. **최종 채널** — 웹 발표? 내부 노션 임베드? 공개 URL?
5. **배포 필요 여부** — 로컬 사용? Netlify 배포?
6. **민감 정보 포함 여부** — 비밀번호·고객 개인정보 등

원본 자료가 있으면 **먼저 전부 파싱**한다. CSV는 Read, URL은 WebFetch 또는 Chrome MCP로 읽어 구조 파악 후 합의.

#### 1-2. 대략적 챕터 구성 합의

콘텐츠 세부는 아직 쓰지 않고, **챕터 목록과 순서만** 합의한다.

예시:
```
1. Welcome (환영)
2. 업계 & 포지셔닝
3. 회사 소개
4. 조직도
5. 일하는 방식
6. 회의 & 보고
7. Operations
```

> 서브 슬라이드 분할·PPT 강조 포인트는 이 단계에서 확정하지 않는다. Phase 2에서 실제 콘텐츠를 써본 뒤 자연스럽게 결정됨.

---

### Phase 2. MD 파일 채우기

확정된 챕터별로 **파트 1개씩 초안 → 피드백 → 확정 → MD 저장** 루프를 돌린다.
절대 전체를 한 번에 뽑지 말 것.

#### 2-1. 대화 루프

1. 해당 파트 초안을 **마크다운 코드블록**으로 제시
2. 확인 요청 2~3개 명시 (빠진 정보, 표현 선택지 등)
3. 사용자 피드백 반영
4. 확정되면 바로 `content/XX_<name>.md` 파일로 **저장**
5. 다음 파트로 이동

**중복·과시 경계**: 매출·달성 수치는 **한 슬라이드에만** 노출. 여러 슬라이드에 같은 숫자를 반복하면 사용자가 민망해함 (경험칙).

#### 2-2. 저장 구조

```
<project>/
├── content/
│   ├── 00_index.md              ← 전체 목차·메타 정보
│   ├── 01_welcome.md            ← 챕터 1
│   ├── 02_industry.md           ← 챕터 2
│   └── ...
```

파일명은 **숫자 접두어 + 주제** 규칙. 이후 `sections/*.html`과 1:1 매칭되어 웹 생성 시 혼란 없음.

#### 2-3. 각 챕터 MD 규격 (엄수)

```markdown
---
chapter: 2
title: 업계 & 포지셔닝
slides: 3
source: https://example.com/original-doc   # 원본 자료 출처 (선택)
notion_link: https://notion.so/...          # 관련 노션 링크 (선택)
---

# 업계 & 포지셔닝

## 슬라이드 1 — 대행사 정의
...본문 (표·리스트·인용 자유)...

## 슬라이드 2 — 대행사의 본질
...

## 슬라이드 3 — 스마트브랜딩의 포지셔닝
...
```

- YAML frontmatter: chapter·title·slides·source·notion_link
- `## 슬라이드 N — 제목` 헤딩으로 서브 슬라이드 경계 명확히
- 히어로 강조는 `> **"..."**` 또는 `**대형 키워드**`로 표시 (Phase 3에서 웹에 반영)

#### 2-4. `00_index.md` 규격

```markdown
---
project: <프로젝트명>
target: <대상 독자>
purpose: <목적 한 줄>
created: YYYY-MM-DD
---

# 목차

| # | 챕터 | 슬라이드 수 | 파일 |
|---|---|---|---|
| 1 | Welcome | 1 | 01_welcome.md |
| 2 | 업계 & 포지셔닝 | 3 | 02_industry.md |
| ... |

## 원본 자료
- ...

## 디자인·톤 지시사항
(해당 시)
```

#### 2-5. 구조 확정 (Phase 2 종료 시점)

모든 챕터 MD가 채워지면, 이 시점에서 **서브 슬라이드 분할·PPT 강조 포인트**를 최종 확정한다. 콘텐츠 기반이라 결정이 훨씬 쉬움.

참고 표 (경험치 기반 서브 슬라이드 수):

| 주제 성격 | 서브 슬라이드 | 이유 |
|---|---|---|
| 환영/표지 | 1 | 메시지 하나만 |
| 개념·정의 | 2~3 | 정의→예시→적용 |
| 회사 소개·포지셔닝 | 2~3 | 정보·서비스·차별점 |
| 조직도·명단 | 1~2 | 계층 + 좌석 등 |
| Culture·룰 | 2 | 원칙·채널 |
| Operations·다항목 | 4~6 | 항목별 1장 |

Phase 2 완료까지는 웹 코딩 착수 X.

### Phase 3. 웹앱으로 구현 (파트별 분할)

상세 스펙은 **`references/web-spec.md` 참조**. 핵심:

- 저장 경로: `<project>_web/`
- 파일 구조: `src/shell.html` + `sections/01_*.html` ~ `XX_*.html` + `css/styles.css` + `js/app.js` + `build.bat` + `build.sh` + `README.md`
- 기술 스택: Vanilla HTML/JS + Tailwind CDN, `file://` 더블클릭 호환
- **단일 index.html 지양**. 파일 분할 + 빌드 스크립트로 `index.html` 재생성
- 디자인: 블루 팔레트, 16:9 반응형, 좌측 사이드바 탭 + 하단 화살표 서브슬라이드 + 키보드 ← →
- PPT 타이포: 타이틀 `clamp(32~56px)`, 히어로 `clamp(48~96px)`, 본문 `clamp(16~20px)`
- content-card 간격은 **부모 `gap`으로만** 부여 (`.child + .child { margin }` 금지)
- 외부 자료 링크는 `.notion-link-btn` 스타일

**Phase 2의 `content/*.md`를 source of truth로 사용**한다. coder 에이전트에 위임할 때 MD 파일 경로를 함께 전달.

구현은 `coder` 에이전트 위임, 직접 코딩 X.

### Phase 4. 세부 디자인 피드백

스크린샷·구두 피드백 기반 반복 개선:

- 사용자가 스크린샷을 붙이면 → 해당 슬라이드만 수정
- 레이아웃 이슈는 `references/web-spec.md`의 "흔한 이슈" 섹션 먼저 체크
- 동일 이슈 3회 이상 반복 → 전역 CSS 규칙 의심, 루트 수정
- **내용 변경은 MD 먼저 수정 → 웹 재생성** (MD가 source of truth이므로)

---

### Phase 5. 웹앱 게시 (배포)

상세 절차는 **`references/deploy-netlify.md` 참조**. 핵심:

- **배포 전 필수 체크**: 민감 정보(비밀번호·개인정보·내부 링크) 스캔. 발견 시 사용자에게 처리 방향 확인 (제거 / 노션 링크 대체 / 비밀번호 보호 호스팅)
- Netlify Drop (https://app.netlify.com/drop): 폴더 드래그 → 즉시 URL
- 불필요 파일(`src/`, `sections/`, `build.*`, `*.bak`, `content/`) 제외 권장

## 팀원 사용 안내

이 스킬은 다음과 같은 자연어로 트리거된다:

- "발표 자료를 웹으로 만들어줘"
- "이 CSV를 탭형 웹페이지로"
- "보고서를 발표용 웹 슬라이드로"
- "16:9 슬라이드 형식으로"
- "소개서 웹 버전 생성"

트리거되면 **Phase 1부터 순차 진행**한다. 중간 단계부터 시작하면 맥락이 비어서 품질이 떨어짐.
