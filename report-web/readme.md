# report-web 사용 가이드 (팀원용)

발표·보고·공유용 웹 자료를 AI와 함께 단계별로 만드는 스킬입니다.

원본 자료(CSV·노션·회의록·메모)만 있으면, 대화를 통해 정리하고 16:9 탭형 웹 슬라이드로 완성한 뒤 Netlify에 배포까지 해줍니다.

---

## 언제 쓰면 좋은가

- 신규 직원 **온보딩 자료**를 웹 페이지로 만들고 싶을 때
- 클라이언트용 **보고 자료**를 URL로 공유하고 싶을 때
- **사내 교육·소개서**를 PPT 대신 웹 슬라이드로
- 프로젝트·분기 **리뷰 자료**를 탭으로 나눠 공유하고 싶을 때
- 기획안·스프레드시트를 **발표용 시각 자료**로 변환하고 싶을 때

---

## 준비물

무엇이든 있는 대로 준비하면 됩니다. 많을수록 좋지만 다음 중 하나만 있어도 시작 가능:

- 원본 자료 (CSV / 엑셀 / 노션 페이지 URL / 회의록 / 워드 문서 / 손글씨 메모 사진)
- 대상 독자 (누구에게 보여줄 자료인가)
- 주제 한 문장

**민감 정보(비밀번호·고객 개인정보 등)가 포함됐는지** 미리 체크해주세요. 공개 URL에 올라가면 안 되는 정보는 배포 전 단계에서 처리합니다.

---

## 진행 흐름 (5단계)

### Phase 1. 발표 자료 기획
AI가 대상 독자·원본 자료·주제·배포 필요 여부 등을 묻습니다. 답하면서 **챕터 목록**까지 합의.

### Phase 2. MD 파일 채우기
AI가 **한 챕터씩** 초안을 제시하면, 피드백 주면서 확정. 확정된 챕터는 `content/XX_<주제>.md` 파일로 저장됩니다. 이 MD가 **원본 보관용**이자 이후 웹 생성의 재료가 됩니다.

> 한 번에 전체를 뽑지 않고 파트별로 돌립니다. 시간은 좀 더 걸리지만 결과 품질이 확 달라져요.

### Phase 3. 웹앱 구현
확정된 MD를 바탕으로 AI가 16:9 웹페이지를 생성. 파일은 섹션별로 분리돼 있어 나중에 특정 슬라이드만 쉽게 수정 가능합니다.

### Phase 4. 디자인 피드백
스크린샷 붙여 넣고 "이 부분 이렇게 고쳐줘" 하면 반영. 반복.

### Phase 5. 배포
민감 정보 스캔 후 Netlify Drop으로 공개 URL 발급. 5분 안에 완료.

---

## 예상 소요 시간

| 단계 | 소요 |
|---|---|
| Phase 1 기획 | 5~10분 |
| Phase 2 MD 작성 | 30~90분 (자료량에 따라) |
| Phase 3 웹 구현 | 5분 (AI 자동) |
| Phase 4 피드백 | 15~30분 (세부 수정에 따라) |
| Phase 5 배포 | 5분 |

총 1~3시간 안에 깔끔한 웹 발표 자료 완성 가능.

## 자주 묻는 질문

### Q. 중간에 내용을 바꾸고 싶으면?
A. `content/XX_*.md` 파일을 직접 수정하거나 AI에게 말하세요. MD가 원본이라 이걸 고치면 웹도 재생성됩니다.

### Q. 웹페이지를 나중에 업데이트하려면?
A. MD 수정 → `build.bat` 실행 → 생성된 `index.html`을 Netlify에 재드롭.

### Q. 민감 정보가 있는데 공개해도 되나요?
A. 안 됩니다. Phase 5에서 AI가 스캔하고 제거/노션 링크 대체 방식을 제안할 거예요. 내부 직원만 접근해야 하면 Cloudflare Access 같은 비밀번호 보호 호스팅을 권장합니다.

### Q. 파일을 어디다 저장하나요?
A. AI가 새 폴더를 만들어줍니다 (예: `onboarding_web/`, `q1_report_web/`). 경로는 Phase 1에서 합의해요.

### Q. 혼자 써도 되나요?
A. 물론. 원본 자료 준비 → AI에게 요청 → 대화하면서 완성.

## 문제 해결

**스킬이 자동 실행 안 될 때**
- `/report-web`을 명시적으로 타이핑
- "만들어줘" 같은 실행 지시어 사용

**디자인이 마음에 안 들 때**
- Phase 4에서 스크린샷 + 구체 피드백 반복
- "이 색으로", "여기 글씨 더 크게" 등 구체적 요청이 효과적

**배포 후 업데이트**
- MD 수정 → 재빌드 → Netlify 재드롭
